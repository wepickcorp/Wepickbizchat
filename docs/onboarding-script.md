# 인앱 온보딩 스크립트 — 첫 발송까지 5분 (미용실 사장님)

> 셀프서비스의 "진짜 세일즈" = 가입 → **첫 발송 완료(아하 모먼트)**.
> 대상: 처음 온 미용실 사장님 | 톤: 당근마켓식 친근한 반말체
> 연결: [sales-playbook.md](sales-playbook.md) · [launch-strategy-1pager.md](launch-strategy-1pager.md) · 측정 이벤트는 [event-logging-design.md](event-logging-design.md)
> 상태: **기획 초안** (코드 아님 — 카피/플로우 스펙)

---

## 핵심 원칙 4가지
1. **빈 화면 금지** — 항상 미용실용 검수 템플릿에서 시작.
2. **결제 공포 < 아하 모먼트** — *돈 내기 전에* 첫 발송 경험을 만든다. → **체험 크레딧 필수**(아래 §결정 참고).
3. **한 화면 = 한 결정** — 사장님은 한 번에 하나만 고른다.
4. **숫자로 안심** — "약 OO명에게 / OO원" 을 항상 눈앞에 보여준다.

---

## ⭐ 선결정: 체험 크레딧 (가장 중요한 레버)
지금 플로우는 **발송 직전에 결제**가 끼어 있어 아하 모먼트(첫 발송+리포트) *앞에* 돈 장벽이 있다. 부트스트랩 셀프서비스에서 이게 1번 누수가 될 가능성이 높다.
- **권장**: 가입 시 **체험 크레딧 200C(=100명 발송분)** 지급 → 결제 없이 첫 발송까지 도달.
- 효과: "공짜로 한 번 보내보고, 리포트 보고, 그다음 충전" (Mailchimp/Duolingo식 *가치 먼저*).
- 100명은 동네 1차 테스트로 충분 & 비용 통제 가능.

---

## 화면별 스크립트

### S0. 가입 (auth)
- **목표**: 마찰 최소화. 가입 자체가 목적이 아님.
- **보이는 것**: 이메일/비번(또는 소셜). 사업자번호·회사명 등 *나중에*.
- **카피**: "1분이면 시작해요. 가게 정보는 첫 발송 때 받을게요."
- **이탈방지**: 가입 단계에서 사업자 인증·결제정보 절대 안 받음.
- **이벤트**: `signup_completed` (서버)

### S1. 환영 + 한 질문
- **목표**: 업종 잡고 개인화 시작. (Canva식 "용도 선택")
- **보이는 것**: "어떤 손님을 모으고 싶으세요?" → [신규 손님] / [재방문(단골)] + 업종 칩(미용실 선택)
- **카피**: "사장님 가게에 딱 맞게 준비해드릴게요. 3개만 누르면 끝나요."
- **이탈방지**: 객관식만. 자유입력 0.
- **이벤트**: `onboarding_started` (클라)

### S2. 템플릿 갤러리 (빈 화면 금지)
- **목표**: "글 안 써도 된다"를 즉시 체감.
- **보이는 것**: 미용실용 **검수 완료 문자 카드 3~5개** (재방문 쿠폰형 / 신규 할인형 / 신메뉴·시술 안내형). 각 카드에 실제 문구 미리보기.
- **카피**: "이 중에 마음에 드는 문구 하나 골라요. 사장님은 가게 이름만 넣으면 돼요."
- **이탈방지**: 추천 1개를 상단에 "인기" 배지로 띄워 결정 부담 ↓.
- **이벤트**: `template_selected` (클라, templateId)

### S3. 가게 정보 채우기 (변수 입력)
- **목표**: 최소 입력으로 문자 완성.
- **보이는 것**: 고른 템플릿에서 `{가게명}`,`{할인}` 같은 변수만 입력칸으로. 입력하는 즉시 **문자 미리보기 실시간 갱신**.
- **카피**: "가게 이름이랑 혜택만 넣어볼까요? 오른쪽에 실제 가는 문자가 바로 보여요."
- **이탈방지**: 변수 2~3개 이하. 기본값 미리 채워둠.
- **이벤트**: `template_filled` (클라)

### S4. 받을 고객 (타겟)
- **목표**: 막막한 타겟팅을 "버튼 몇 개"로.
- **보이는 것**: 우리 가게 반경(슬라이더), 연령/성별 칩. 조건 바꿀 때마다 **"약 OO명에게 갈 수 있어요" 실시간**.
- **카피**: "우리 가게 반경이랑 손님 연령만 골라요. SK텔레콤 데이터라 동네 손님 정확해요."
- **이탈방지**: 미용실 기본 추천 타겟(반경 1.5km·여성·20~40대 등) 미리 세팅 → 그냥 "다음" 가능.
- **이벤트**: `targeting_completed` (클라, targetCount)

### S5. 발송 확인 (결제 공포 구간 ★)
- **목표**: 마지막 불안 제거 + 테스트로 신뢰.
- **보이는 것**: ①최종 문자 미리보기 ②"**○○명에게 / OO크레딧 차감(=남은 크레딧)**" ③[내 번호로 테스트 1건 보내기] 버튼 ④[발송하기]
- **카피**: "보내기 전에 사장님 폰으로 딱 1건 먼저 받아볼래요? 진짜 어떻게 가는지 확인하고 보내요."
- **이탈방지**: 테스트 발송 = "잘못 갈까 봐"의 공포 해소. 체험 크레딧 있으면 여기서 결제 없이 [발송하기] 활성화.
- **이벤트**: `send_test_clicked`, `send_confirm_viewed` (클라)

### S6. 크레딧 충전 (체험 크레딧 소진 후 / 부족 시에만)
- **목표**: 첫 결제 공포 최소화.
- **보이는 것**: "라이트 10만 원 = 1,000명" 강조(입문용), 부족분만 명확히. KISPG 카드결제.
- **카피**: "이번 발송엔 OO크레딧이 필요해요. 라이트로 충전하면 1,000명까지 보내고, 안 쓰면 1년 남아요."
- **이탈방지**: 패키지는 라이트를 디폴트 추천(결정 부담 ↓). 환불 가능 문구 노출.
- **이벤트**: `credit_purchase_initiated`(클라) → `credit_purchased`(서버, KISPG 콜백)

### S7. 발송 완료 = 아하 모먼트 🎉
- **목표**: 성취감 + 다음 행동 2개 심기(리포트 기대 / 추천).
- **보이는 것**: "○○명에게 발송 완료!" 축하 + "며칠 뒤 방문 늘었는지 리포트로 보여드릴게요" + **추천 카드**.
- **카피**: "첫 광고 보냈어요, 사장님 🎉 / 주변에 단골 늘리고 싶은 사장님 있으면 소개해줘요 — 두 분 다 크레딧 드려요."
- **이벤트**: `campaign_sent`(서버) = **활성화 지표** / `referral_shared`(클라)

### S8. (D+3) 리포트 넛지 — 리텐션 시작
- **목표**: 재방문 → 재발송 루프 시작.
- **보이는 것**: 푸시/문자: "사장님, 지난 광고 결과 나왔어요. 도달 OO명! 또 보낼까요?"
- **카피**: "반응 좋았어요. 이번 주말 손님 부를 문자 하나 더 보낼까요?"
- **이벤트**: `report_viewed`, (재발송 시) 2번째 `campaign_sent`

## 퍼널·이벤트 매핑 (한눈에)
```
S0 signup_completed → S2 template_selected → S4 targeting_completed
 → S6 credit_purchased → S7 campaign_sent(★활성화) → S8 재발송
```
- 1-pager §3 퍼널의 각 칸 = 위 이벤트. 어느 화면(S?)에서 가장 많이 죽는지 = 다음 개선 1순위.
