## 1. 문제 (Why)

비즈챗은 "전문가용 복잡한 광고 운영 툴"에서 **"동네 사장님 누구나 5분 만에 직접 보내는 광고 도구"**로 전환했다.
제품은 바뀌었지만, **사장님이 가입 후 실제로 첫 발송까지 도달하는지**를 우리는 아직 모른다.

> 핵심 가정: *"광고는 어렵고 무섭다"* 는 사장님의 공포가, 가입과 첫 발송 사이 어딘가에서 이탈을 만든다.

---

## 2. 핵심지표 (North Star)

> **주간 첫 발송 완료 사장님 수 (Weekly Activated Senders)**
> 보조지표: **가입 → 첫 발송 완료 전환율**, 활성화 속도(가입→첫 발송 시간)

- 활성화 정의 = `credit_ledger.type='use'` 1건 이상 (= 크레딧이 실제 발송에 차감됨)
- "가입자 수"·"매출"은 후행지표로만 본다. **첫 발송을 끝낸 사장님만이** 재충전·리텐션·입소문의 출발점.

## 4. 가장 큰 누수 (The Leak)

> 프로덕션 Q3에서 전환율이 가장 낮은 칸 = 이번 2주의 유일한 전쟁터.

로컬 신호 기준 **2개 후보**:

- **(A) 가입 → 캠페인 생성 (~15%)**: 가입자 대부분이 캠페인을 *시작도 안 함*. → 빈 화면/막막함 문제.
- **(B) 크레딧 충전 → 발송 완료 (~0%)**: 충전하고도 발송을 안 함. → 첫 발송 직전 공포/막힘.
  (Q4 `draft`·`approval_requested` 상태에 캠페인이 고여있는지 교차 확인)

→ **프로덕션 숫자로 A/B 중 하나를 확정**하고 나머지는 이번 주기엔 건드리지 않는다.

## 5. 가설 & 실험 (이번 2주, 1개만)

> 누수가 **(B) 충전→발송**으로 확정될 경우의 예시. (A)면 실험 내용만 교체.

- **가설**: 사장님은 *"이 돈 내고 보냈는데 잘못 가면 어쩌지"* 때문에 발송 버튼을 못 누른다.
  발송 직전에 **"누구에게 / 몇 명에게 / 얼마 차감"**을 한 화면에 명확히 보여주고,
  **무료 테스트 발송(본인 번호로)** 을 끼워 넣으면 첫 발송 전환율이 오른다.
- **변경**: 발송 확인 화면 리디자인 + "내 번호로 1건 테스트 발송" 버튼 추가.
- **성공 기준**: 충전→발송 전환율 `[현재 %]` → **+15%p**, 활성화 속도(중앙값) 단축.
- **측정**: `launch-funnel.sql` Q3·Q5 를 매일 실행해 비교. (이벤트 로깅 도입 후 실시간 대시보드로 이관)
